Eclipse Test Integration "2.0"

Unofficial fork of Eclipse JDT JUnit plugin. This fork aims to provide a better visual experience of failed test results.

Download this project as a .zip file Download this project as a tar.gz file

Features

Currently there are implemented 2 features

  • Showing ignored / skipped tests in Eclipse View
  • Showing nice comparison messages for some assertj failures.

How to try it?

  1. Clone the source code from

    https://github.com/azachar/assertj-core/tree/1_6_0_dev_fork_DescriptiveAssertionException

  2. Compile assertj-core 1.6.0-snapshost by

    mvn clean install

  3. Clone the source code of the patched Eclipse plugins from

    https://github.com/azachar/org.eclipse.jdt.ui from the default branch (R4_3_1_all-features-merged)

  4. Ensure that you are running Eclipse PDE Kepler SR-1.

    Import the following projects into your workspace

    org.eclipse.jdt.ui/org.eclipse.jdt.junit

    org.eclipse.jdt.ui/org.eclipse.jdt.junit.core

    org.eclipse.jdt.ui/org.eclipse.jdt.junit4.runtime

    Install these 3 plugins into your Eclipse Kepler by

    a) Select the mentioned projects above and from the context menu choose Export -> Deployable plug-ins and fragments

    b) Choose Install into host. Repository

    c) Press Finish button and wait until it will be installed into your Eclipse Workspace Only (which enables you to test it).

  5. Use assertJ's assertThat(a).isEqualToComparingFieldByField(b) or simply import to your Eclipse this sample project and run as your junit project within your Eclipse.

    https://github.com/azachar/assertj-core/tree/1_6_0_dev_fork_DescriptiveAssertionException/example

Now you are able to see the nice comparison of failed fields in your junit view called "Bug Trace" that shows a quick overview of failed tests elements.

Also in the stacktrace for some assertj exception (initially for comparison field by field of two objects) you click on the first line in the stacktrace of your failed test to open "visual" comparision.

Beta

In the branch https://github.com/azachar/org.eclipse.jdt.ui/tree/R4_3_1-bug-trace-beta you can find the latest release of the pretty diff based visualization of failures.
