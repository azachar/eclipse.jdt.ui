Eclipse Test Integration "2.0"

This fork of Eclipse JDT feature aims to provide a better visual experience of failed test results.

View project onGitHub

New Features

  • Elimination of search for a failure in the stack trace
  • Shows failures clearly in the new inline Bug Trace viewer
  • Shows colored differences between expected and actual values
  • Ability to filter @Ignored tests
  • Some complex failures from the AssertJ project (fork is provided here) are visualized in the inline Bug trace view.

Installation into Eclipse Kepler Workspace

  1. Clone the source code of the forked Eclipse Kepler 4.3.1 from 

    git clone -b master https://github.com/azachar/org.eclipse.jdt.ui

  2. Ensure that you are running Eclipse PDE Kepler SR-1.

    Import the following projects into your workspace 

    org.eclipse.jdt.ui/org.eclipse.jdt.junit

org.eclipse.jdt.ui/org.eclipse.jdt.junit.core

org.eclipse.jdt.ui/org.eclipse.jdt.junit4.runtime

    Install these 3 plugins into your Eclipse Kepler by

    a) Select the mentioned projects above and from the context menu choose Export -> Deployable plug-ins and fragments

    b) Choose Install into host. Repository

    c) Press Finish button and wait until it will be installed into your Eclipse Workspace.

Now you are able to see the clearly failures in your junit view called ****Bug Trace**** that shows a quick overview of failed tests elements.

AssertJ with colored failures visualization

I have modified AssertJ project in order to provide more clear visualization of failures. Currently are supported only following asserts:

assertThat(a).isEqualToComparingFieldByField(b)

Installation of the AssertJ fork with Comparisons

  1. Clone the source code from 

    git clone -b 1_6_0_dev_fork_DescriptiveAssertionException https://github.com/azachar/assertj-core

  2. Compile assertj-core 1.6.0-snapshost by 

    mvn clean install

  3. Use assertJ's assertThat(a).isEqualToComparingFieldByField(b) or simply import to your Eclipse this sample project and run as your junit project within your Eclipse.

    assertj-core/example

Also in the JUnit stacktrace view for some AssertJ exception (initially for comparison field by field of two objects) you click on the first line in the stack trace of your failed test to open "visual" comparison.
